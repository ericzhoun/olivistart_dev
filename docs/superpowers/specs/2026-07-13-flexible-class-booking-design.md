# Flexible Credit-Based Class Booking

Date: 2026-07-13

## Problem

Today a student enrolls into exactly one fixed weekly `class_schedule` and buys a bulk pack of classes for that slot only. There's no way for a student to attend a different day/time when their usual slot doesn't work, and no per-date attendance or capacity tracking — `max_seats` is checked only against the recurring slot as a whole, not against any specific date.

## Goals

- Student picks a regular ("home") weekly slot at enrollment time, same as today, and buys a pack of N credits.
- Home-slot attendance happens automatically each week — no per-week booking action required from the student.
- Student can additionally book a make-up session at a different slot within the same program when their home slot doesn't work that week.
- Student can mark in advance that they'll skip an upcoming home-slot date, freeing that seat.
- Attendance (present/no-show) is marked by admin/teacher per specific date; only a marked "present" consumes a credit.
- Credits never expire and can go negative (admin handles overages manually, outside the system).
- Make-up bookings and skips require at least 24h notice.

## Non-goals

- No automatic credit expiry or semester-scoped credit forfeiture.
- No system-enforced blocking when credits run out — admin follows up manually.
- No restriction of make-up eligibility by age group — any slot in the same program is eligible.
- No cutoff-based partial-refund/no-charge logic for cancellations (a no-show simply never gets marked "attended," so it never costs a credit — no separate cancellation-fee logic needed).

## Data Model

### `class_sessions` (new)

One row per actual calendar date a schedule meets.

| field | type | notes |
|---|---|---|
| id | uuid | |
| schedule_id | fk → class_schedules | |
| class_date | date | |
| status | enum: `scheduled`, `cancelled` | admin can cancel a whole date (e.g. weather) |

Generated by expanding `class_schedules.day_of_week` across the owning semester's `start_date`–`end_date` whenever a schedule is created, or when a semester's date range is extended. If a schedule's date range is extended after students are already enrolled in it, admin must trigger a backfill of `home` bookings for those enrollments (see Flows below) — this is a deliberate manual step, not automatic.

### `bookings` (new)

One row per student per session. Drives both home-slot attendance and make-up bookings through a single mechanism.

| field | type | notes |
|---|---|---|
| id | uuid | |
| enrollment_id | fk → enrollments | |
| session_id | fk → class_sessions | |
| type | enum: `home`, `makeup` | |
| status | enum: `scheduled`, `attended`, `no_show`, `skipped`, `cancelled` | |
| booked_at | timestamp | |
| marked_at | timestamp, nullable | when attendance was recorded |
| marked_by | fk → admin user, nullable | |

Status semantics:
- `scheduled`: expected to attend (default state for both home and make-up bookings).
- `attended`: admin confirmed presence. **Only this status deducts a credit.**
- `no_show`: admin confirmed absence. Does not deduct a credit.
- `skipped`: student proactively opted out of a home-slot date in advance (≥24h). Does not deduct a credit. Frees the seat.
- `cancelled`: student cancelled a make-up booking in advance (≥24h). Does not deduct a credit. Frees the seat.

### `enrollments` (existing, extended semantics)

No new required columns beyond what exists (`schedule_id`, `num_classes_enrolled`, `status`, `registration_complete`, student fields). `schedule_id` is now understood as the **home slot**. Credit balance is computed, not stored:

```
credits_remaining = num_classes_enrolled - count(bookings WHERE enrollment_id = X AND status = 'attended')
```

Can be negative.

### Capacity

For any given session, open seats:

```
open_seats = class_schedules.max_seats - count(bookings WHERE session_id = X AND status IN ('scheduled', 'attended'))
```

## Flows

**Enrollment.** Unchanged UI: student picks a home slot and buys a pack of N classes. On payment confirmation, the system creates `home` booking rows (`status = scheduled`) for every existing future `class_session` of that schedule.

**Home-slot attendance.** No student action needed weekly. Admin opens a session date in the admin panel and marks each expected student `attended` or `no_show`. Only `attended` deducts a credit.

**Skip a home-slot date.** Student selects an upcoming home-slot date from their account (≥24h out) and marks it as skipped. Flips that one booking row to `skipped`, freeing the seat.

**Book a make-up.** Student browses sessions across any schedule in the same program (any age group) with open seats, at least 24h out. Booking creates a new `makeup` booking row against that session. Does not touch the student's home-slot rows.

**Cancel a make-up.** Same 24h cutoff. Flips the row to `cancelled`.

**Credit exhaustion.** No enforcement. Balance can go negative; visible to admin for manual follow-up.

**Schedule/semester date-range extension.** Admin action that generates additional `class_sessions` rows, and must also backfill `home` booking rows for students already enrolled in that schedule.

## UI Changes

**Account page (`/account`).** Each enrollment card gains:
- Credit balance display (negative shown distinctly).
- "My upcoming classes" list merging home-slot sessions and make-up bookings, sorted by date, each with a "Skip"/"Cancel" action disabled inside the 24h window.
- "Book a make-up class" — a calendar (reusing the existing `CalendarView` pattern) scoped to the enrollment's program, showing only sessions with open seats.

**Admin (`/admin`).** New "Sessions" view listing upcoming `class_sessions` per schedule. Selecting a date opens an attendance sheet listing everyone with a `scheduled` booking for that session (home and make-up bookings both shown, labeled by type), with one-click Attended/No-show actions. This extends the current `admin/enrollments` page, which today only shows pack-level status, not per-date attendance.

**Enroll flow (`/art-class/enroll/[scheduleId]`).** Structurally unchanged. Copy updated to frame the chosen slot as the student's *regular* slot, with make-ups available elsewhere, rather than a full lock-in.

## Error Handling

- Booking/skip/cancel actions outside the 24h cutoff are rejected server-side (not just hidden in the UI) to prevent race conditions near the cutoff boundary.
- Make-up booking is rejected if the target session has no open seats at request time (re-checked server-side at booking, not just from a possibly-stale client view).
- Marking attendance on a booking that's already `attended`/`no_show` is idempotent — re-marking simply updates the status rather than creating duplicate credit deductions (deduction is derived from current status count, not an incrementing counter).

## Testing

- Unit-level: credit balance computation, open-seat computation, and 24h cutoff boundary logic (each as pure functions over booking/session data).
- End-to-end: enroll → verify home bookings generated → admin marks attendance → credit deducts; student books make-up → capacity decrements → student cancels make-up before cutoff → capacity restored; student attempts a skip/make-up/cancel inside the 24h window → rejected.
